# Enemy Spawn System – 기획자 설명 문서

---

# 1. 시스템 목적

이 시스템은 **적을 코드 수정 없이 자유롭게 배치하기 위한 구조**입니다.

기획자는 다음을 직접 제어할 수 있습니다:

- 어떤 몬스터가 등장하는지
- 몇 초 뒤에 등장하는지
- 어디에서 등장하는지
- 스테이지별 등장 패턴

개발 코드 수정 없이 데이터만 변경하여 조정 가능합니다.

---

# 2. 시스템 핵심 개념

이 시스템은 두 가지 데이터로 움직입니다.

1) 몬스터 정의 데이터  
2) 소환 스케줄 데이터  

---

# 3. 전체 구조 흐름

```
몬스터 정의 데이터
        ↓
몬스터 ID 자동 등록
        ↓
스폰 스케줄 데이터
        ↓
시간 조건 충족 시 적 생성
```

---

# 4. 몬스터 정의 데이터 (EnemyConfig)

각 몬스터는 다음 정보를 가지고 있습니다:

- 몬스터 고유 ID
- 연결된 프리팹(실제 게임 오브젝트)

### 기획자가 관리하는 부분

- 몬스터 ID
- 어떤 프리팹을 사용할지

### 특징

- 몬스터 이름과 무관
- ID만으로 관리
- 새로운 몬스터 추가 시 코드 수정 필요 없음

---

# 5. 스폰 스케줄 데이터 (SpawnSchedule)

이 데이터는 **"언제, 무엇을, 어디에" 등장시킬지**를 정의합니다.

각 스폰 이벤트는 다음 정보를 가집니다:

- monsterId
- spawnTime (몇 초 뒤 등장)
- spawnPosition (등장 위치)

---

# 6. 실제 동작 방식

예시:

| 시간 | 몬스터 | 위치 |
|------|--------|------|
| 2초  | ID 1   | (0,0,0) |
| 5초  | ID 2   | (3,0,0) |

게임이 시작되면:

1. 몬스터 목록이 자동 등록됨
2. 시간이 흐름
3. 지정된 시간이 되면 해당 ID 몬스터가 생성됨

---

# 7. 기획자가 할 수 있는 작업

## 1) 몬스터 추가

- 새 프리팹 제작
- 몬스터 ID 지정
- 스폰 스케줄에 ID 추가

코드 수정 없음

---

## 2) 등장 시간 조정

spawnTime 값만 수정

---

## 3) 등장 위치 조정

spawnPosition 값만 수정

---

## 4) 스테이지별 패턴 제작

Stage1Schedule  
Stage2Schedule  
BossSchedule  

각 스테이지마다 다른 스폰 데이터 사용 가능

---

# 8. 난이도 설계 예시

EasySchedule  
NormalSchedule  
HardSchedule  

난이도에 따라 다른 스폰 데이터를 연결할 수 있음.

---

# 9. 확장 가능 요소

향후 확장 시 기획에서 제어 가능한 요소:

- 웨이브 단위 등장
- 특정 몬스터 동시 다수 등장
- 확률 기반 랜덤 등장
- 랜덤 위치 등장
- 보스 등장 이벤트
- 조건 기반 등장 (플레이어 점수, 체력 등)

---

# 10. 장점 정리

- 코드 수정 없이 콘텐츠 제작 가능
- 스테이지 설계 자유도 높음
- 밸런스 조정 빠름
- 테스트 및 반복 작업 용이
- 대규모 몬스터 추가에도 안정적

---

# 11. 한 줄 요약

이 시스템은  
**"기획 데이터만 수정하면 적 배치가 바뀌는 구조"** 입니다.

개발 개입 없이 반복적인 밸런스 조정이 가능합니다.
